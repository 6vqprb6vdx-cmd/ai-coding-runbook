Standard AWS-style contributing guide for the Bedrock Edition of the Prompt Engineering Interactive Tutorial.

**Reporting bugs / feature requests.** Use GitHub issue tracker. Check for existing issues first. Useful details: reproducible test case or steps, version of code, modifications made, environment/deployment specifics.

**Pull requests.** Work against latest source on `main`. Check existing/recently merged PRs. Open an issue to discuss significant work first.

**PR steps.** Fork the repo. Modify only the files relevant to the change (avoid noisy reformatting that obscures the change). Ensure local tests pass. Commit to fork with clear messages. Send a PR and answer the PR template questions. Follow up on automated CI failures and reviewer comments.
